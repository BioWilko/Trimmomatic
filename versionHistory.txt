Version 0.41: Fix: Bug in IlluminaClippingTrimmer calculateMaximumRange
              Feature: Added extensive JUnit tests
              Performance: Major optimizations in BarcodeSplitter (array-based lookups, loop unswitching, char[] comparison) and fix for short reads
              Performance: Optimized Bzip2 compression (buffer pre-allocation) and CompressionFormat detection
              Feature: Moved to Java 25
              Feature: Replaced Jbzip2 with Apache Commons Compress (concatenated streams)
              Feature: Updated dependencies (Mockito, ByteBuddy, JUnit)
              Feature: Added verbose adapter reporting (-verbose flag)
              Feature: Simplified invocation mode with automatic output filenames and default trimming steps
              Performance: Parallel open of paired-end input files using virtual threads
              Performance: FastqRecord quality integer array cached on first access, eliminating repeated allocation per trimmer
              Performance: FastqRecord trim copy constructor stores a view (offset+length) instead of calling substring(), deferring string materialisation to serialisation time
              Performance: FastqSerializer.writeRecord() writes directly to BufferedWriter, removing per-record StringBuilder allocation
              Performance: ThreadedPipeline task queue widened (threads*2 -> threads*8); sleep(100) busy-wait replaced by blocking RejectedExecutionHandler
              Performance: SerializedBlockQueue compression and output queues widened (buffers -> buffers*2)
              Performance: UncompressedBlockData pre-sizes ByteArrayOutputStream from record lengths and exposes internal buffer to avoid the toByteArray() copy
              Performance: IlluminaClippingTrimmer pre-computes packSeqExternal once per read before adapter loops; packPrefixAndSeq() helper eliminates string concatenation in palindrome matching
              Internal: GzipBlockData, Bzip2BlockData, TrimLogRecord converted to records; BlockData sealed interface
              Internal: SerializedBlock manual Future<BlockData> replaced with CompletableFuture, removing synchronized/wait/notifyAll
              Internal: Switch statements converted to switch expressions throughout (CompressionFormat, FastaRecord, IlluminaClippingTrimmer)
              Fix: ILLUMINACLIP adapter lookup now also searches adapters/ subdirectory in the current working directory, fixing simplified invocation when the JAR is not co-located with the adapters/ folder

Version 0.42: Feature: POLYX trimmer — trims homopolymer runs (polyA, polyG, polyT, polyC, polyN) from the 3' end; drops the read if the entire read is poly-X
              Feature: LOWCOMPLEXITY trimmer — drops reads whose Shannon entropy (over A/C/G/T base frequencies, N excluded) is below a configurable minimum
              Feature: UMIEXTRACT trimmer — extracts a UMI of configurable length from the 5' end, appends it to the read name, and trims it from the sequence; supports a custom name separator
              Feature: MAXAMBIG trimmer — drops reads whose fraction of N bases exceeds a configurable maximum
              Feature: FastqRecord name-override view constructor for zero-allocation UMI name tagging
              Feature: Interleaved FASTQ input support (-interleaved flag in PE mode) — a single file containing alternating R1/R2 reads is split into two virtual streams via InterleavedParserPair
              Feature: LONGREADCLIP trimmer — adapter clipping for long reads using Hamming distance; adapters loaded from FASTA, N bases treated as wildcards, both forward and RC orientations checked
              Feature: -longread flag (SE and PE) — skips the expensive 10 000-record phred pre-read and defaults to Phred+33, correct for ONT and PacBio platforms
              Fix: MaximumInformationTrimmer crashed with ArrayIndexOutOfBoundsException for reads > 1000 bp; score arrays now grow dynamically and thread-safely for reads of any length
              Performance: Parser.parseBlock() and InterleavedParserPair now cap block size by both record count (32 768) AND byte volume (64 MB), preventing heap exhaustion on long-read inputs
              Performance: FastqParser phred pre-read capped at 4 MB; early EOF break avoids wasted iterations; -longread skips pre-read entirely
              Build: -version now appends the 7-character git commit hash and a -dirty suffix when uncommitted changes are present (e.g. 0.42+a3f8b21-dirty); JAR filename remains version-only (e.g. trimmomatic-0.42.jar)
              Data: Bundled long-read adapter FASTA files — ONT-LSK108-LSK110.fa (R9.4/R9.4.1), ONT-LSK112.fa (R10.3), ONT-LSK114.fa (R10.4.1/Kit 14), ONT-Rapid.fa (RAD/RBK families), ONT-cDNA.fa (PCS109/PCS114); PacBio-RSII.fa (RS II SMRTbell + C2 primer), PacBio-Sequel.fa (Sequel/SequelII/SequelIIe/Revio/ETK2.0 SMRTbell + C2 primer)
